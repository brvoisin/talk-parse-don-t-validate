Now I have a single, snappy slogan that encapsulates what type-driven design means to me, and better yet, it’s only three words long:
Parse, don’t validate.

Alexis King

Je vais vous parler du mantra "Parse, don’t validate" expliqué dans cet article de Alexis King. Après avoir lu cet article, je me rends compte que beaucoup de bugs ou d'erreurs que je vois au quotidien peuvent être évités et on va voir comment.

L'article est illustré par un exemple concret en Haskell. Je vais vous présenter cet exemple dans un langage plus connus: Python.

            from pathlib import Path
            from typing import TypeGuard, TypeVar, Unpack

            T = TypeVar("T")
            NonEmptyTuple = tuple[T, Unpack[tuple[T, ...]]]


            def head(seq: NonEmptyTuple[T]) -> T:
                return seq[0]


            def get_config_directories() -> NonEmptyTuple[Path]:
                config_dirs_string = os.getenv("CONFIG_DIRS", "")
                config_dirs = tuple(
                    Path(dir)
                    for dir in config_dirs_string.split(",")
                    if dir
                )
                if not is_non_empty(config_dirs):
                    raise ValueError("CONFIG_DIRS cannot be empty")
                return config_dirs


            def is_non_empty(
                seq: tuple[T, ...]
            ) -> TypeGuard[NonEmptyTuple[T]]:
                return len(seq) > 0
